# Behavior
# 行为型设计模式
> 用于描述类与对象怎样的交互和分配职责（一共11个）

1. 责任链(Chain)
2. 命令(Command)
3. 解释器(Interpreter)
4. 迭代器(Iterator)
5. 中介者(Mediator)
6. 备忘录(Memo)
7. 观察者(Observer)
    > 定义了一种一对多的依赖关系，让多个观察者对象同时监听某一个主题对象。这个主题对象在状态发生变化时，会通知所有观察者对象，使它们能够自动更新自己。
8. 状态(State)
9. 策略(Strategy)
    > 它定义了算法家族，分别封装起来，让它们之间可以互相替换，此模式让算法的变化，不会影响到使用算法的客户。
10. 模板方法(Template Method)
11. 访问者(Visitor)
